# Сервіс «репорт вразливості» для навчальних проектів

Вебзастосунок для реєстрації та управління звітами (Report System). Робота виконана в межах лабораторної роботи №1 — Варіант 6.

## Реалізований функціонал
- Створення: введення даних про інцидент з валідацією.
- Редагування: можливість зміни вже створених записів у окремому фреймі.
- Видалення: видалення неактуальних інцидентів з підтвердженням.
- Сортування та фільтрація: сортування за полями; опціональний фільтр за типом або критичністю.
- Збереження: дані зберігаються у localStorage браузера.
- Відображення: динамічний рендер таблиці з thead/tbody і заголовками th.
- Події: делегування подій для таблиці для ефективної обробки динамічних рядків.

---

##  Валідація введених даних
- Ім'я користувача — обов'язкове поле до 10 символів;
- Пріоритет — обовʼязкове поле;
- Статус — не обовʼязкове поле;
- опис інциденту — до 4000 символів.
- UX валідації: адресні повідомлення поруч із полями і підсвічування невалідних полів; сабміт блокується при помилках.

## Запуск проєкту
1. Завантажте репозиторій.

3. Додаткові залежності відсутні.

---

## Короткий чекліст для демонстрації
- Додати запис з валідними даними → запис зʼявляється в таблиці.
- Спроба додати з порожніми або некоректними полями → помилки поруч із полями.
- Редагувати запис → зміни зберігаються і зберігаються після перезавантаження.
- Видалити запис → запис видаляється і localStorage оновлюється.
- Пошук/фільтр/сортування працюють коректно.
- Вставити тестовий XSS пейлоад у поле Опис або Репортер → скрипт не виконується, відображається як текст.
